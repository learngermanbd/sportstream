🏗️ Final Tech Stack

FCM Only Sentry Supabase Self-Hosted Config Render/Railway
🔔

Push: Firebase FCM

ONLY Firebase service used. Free, unlimited. Firebase Messaging dependency + google-services.json. Server-side via FCM HTTP v1 API from your backend.

🐛

Crashes: Sentry

Replaces Firebase Crashlytics. 5K errors/mo free. Better stack traces, performance monitoring, open-source. SDK: io.sentry:sentry-android.

🗄️

Database: Supabase (PostgreSQL)

Managed Postgres with built-in Storage (for logos/thumbnails/videos) and Auth (for admin login). Free tier: 500MB + 1GB storage. $25/mo Pro for production (no pause).

⚙️

Config: Your Own API

Replaces Firebase Remote Config entirely. Your admin panel serves config at /api/config (API URLs, update URLs, Telegram links, feature flags, maintenance mode).

🖥️

Backend: Render → Railway

Node.js + Express API on Render (free for dev). Migrate to Railway ($5/mo) for production — no spin-down, always-on for push notifications and live content.

🎬

Video: External URLs + Supabase

Live streams: admin adds external HLS/DASH URLs. ExoPlayer plays directly. Highlights: upload to Supabase Storage or Cloudflare Stream. No self-hosted video encoding.

💰

Monetization: Hybrid

Free tier: AdMob banner + interstitial ads. Premium ($2-3/mo): remove ads + HD streams + offline downloads. Google Play Billing for subscriptions. Add monetization after gaining users.

📱

Analytics: PostHog (or skip)

Open-source, 1M events/mo free. Self-hostable. Add later when you have real users. Or use Microsoft Clarity for free heatmaps/session recordings.

💡 Pro Tips for Best Results

1
Always Research Before Coding

Use Web Researcher or Docs Researcher BEFORE implementing any feature. This prevents using deprecated APIs or wrong approaches.

2
Use Thinker (Premium) for Complex Logic

For gesture handling, security architecture, data flow design, and tricky bugs - use Thinker (DeepSeek V4 Pro / MiMo 2.5 Pro). Premium models read your files and reason deeply. Save Premium sessions for the hardest tasks only — you get 5 sessions per season.

3
Build After Every Phase

Run ./gradlew assembleDebug after each phase to catch errors early. Don't accumulate build errors across phases.

Review After Significant Changes

Use Code Reviewer after every phase. It catches bugs, style issues, and missing imports that you might miss.

Give Detailed Context in Prompts

When asking Buff (Mini Max M3) to implement something, include file paths, class names, and specific requirements. Vague prompts produce vague code. Mini Max M3 is fast but benefits from detailed prompts — more context = better output.

6
Use Android 6 (API 23) as Baseline

Always mention "works on API 23+" in your prompts. This ensures Buff uses compat libraries and handles version differences.

🚫 Server-Enforced Version Blocking

Older versions CANNOT use the app. The backend API rejects ALL requests from outdated clients with HTTP 426 (Upgrade Required). No data is served. No exceptions.

Mode 1: Force Update (Blocking)

When currentVersion < minAppVersion — the backend API returns HTTP 426 on every request. The app is completely blocked. No events, no channels, no config — nothing works until the user updates.

🔎 Version Comparison Logic

// VersionUtils.kt fun isVersionLower(current: String, target: String): Boolean { val currentParts = current.split(".").map { it.toInt() } val targetParts = target.split(".").map { it.toInt() } for (i in 0 until maxOf(currentParts.size, targetParts.size)) { val c = currentParts.getOrElse(i) { 0 } val t = targetParts.getOrElse(i) { 0 } if (c < t) return true if (c > t) return false } return false }

Supports semantic versioning: 1.0.0, 1.2.3, 2.0.0-beta

// UpdateManager.kt class UpdateManager(private val context: Context) { private val downloadManager = context.getSystemService(DOWNLOAD_SERVICE) as DownloadManager fun downloadUpdate(apkUrl: String): Long { val request = DownloadManager.Request(Uri.parse(apkUrl)) .setTitle("SportStream Update") .setDescription("Downloading new version...") .setNotificationVisibility(VISIBILITY_VISIBLE) .setDestinationInExternalFilesDir(context, DIRECTORY_DOWNLOADS, "update.apk") .setAllowedOverMetered(true) // Allow on mobile data .setAllowedOverRoaming(true) return downloadManager.enqueue(request) } fun observeDownload(downloadId: Long): Flow<DownloadStatus> = flow { while (true) { val cursor = downloadManager.query(Query().setFilterById(downloadId)) if (cursor.moveToFirst()) { val status = cursor.getInt(cursor.getColumnIndexOrThrow(COLUMN_STATUS)) val downloaded = cursor.getInt(cursor.getColumnIndexOrThrow(COLUMN_BYTES_DOWNLOADED_SO_FAR)) val total = cursor.getInt(cursor.getColumnIndexOrThrow(COLUMN_TOTAL_SIZE_BYTES)) when (status) { STATUS_RUNNING -> { val progress = if (total > 0) (downloaded * 100 / total) else 0 emit(DownloadStatus.Downloading(progress, downloaded, total)) } STATUS_SUCCESSFUL -> { val uri = downloadManager.getUriForDownloadedFile(downloadId) emit(DownloadStatus.Completed(uri)) return@flow } STATUS_FAILED -> { emit(DownloadStatus.Failed("Download failed")) return@flow } } } cursor.close() delay(500) // Poll every 500ms } } } sealed class DownloadStatus { data class Downloading(val progress: Int, val downloaded: Int, val total: Int) data class Completed(val uri: Uri) data class Failed(val message: String) }

// ApkVerifier.kt - SHA-256 hash check before install fun verifyApk(context: Context, apkUri: Uri, expectedHash: String): Boolean { val inputStream = context.contentResolver.openInputStream(apkUri) ?: return false val digest = MessageDigest.getInstance("SHA-256") val buffer = ByteArray(8192) var bytesRead: Int while (inputStream.read(buffer).also { bytesRead = it } != -1) { digest.update(buffer, 0, bytesRead) } inputStream.close() val actualHash = digest.digest().joinToString("") { "%02x".format(it) } return actualHash.equals(expectedHash, ignoreCase = true) }

🛠️ Admin Panel Integration

The admin panel controls updates via the Config Editor screen:

Config Editor (Admin Web/App): +--------------------------------------+ | Update Configuration | | ------------------------------------| | Update URL | | [https:///releases/download/v1.2.0/app.apk | | | Latest Version | | [1.2.0] | | | | Minimum Version (force update) | | [1.1.0] | | | | Update Changelog | | [- Fixed stream buffering ]| | [- Added PiP mode ]| | [- Performance improvements ]| | | | [ Save Configuration ] | +--------------------------------------+
Workflow
  • 1. Developer builds new APK version
  • 2. Uploads APK to GitHub Releases (or any hosting)
  • 3. Admin opens Config Editor → sets new latestVersion + updateUrl + changelog
  • 4. All users see "Update available" banner on next app launch
  • 5. When admin bumps minAppVersion → ALL older versions are INSTANTLY blocked by the server. The API returns HTTP 426 on every request. No data is served. Users MUST update to continue.

⚠️ Server-side enforcement means: even if a user disables the update dialog or modifies the APK, they still can't access any API data. The server is the single source of truth.

8-layer defense model inspired by Sportzfy + 12 additional advanced protections

📦

Layer 4: DEX Encryption

APK signature verification, file hash checks, tamper detection. Verify package name, signing certificate at runtime. Matches Sportzfy's m7()/m8() methods.

🔒

Layer 8: SSL Pinning

Layer 11: Secure Key Storage

Layer 12: Network Security
